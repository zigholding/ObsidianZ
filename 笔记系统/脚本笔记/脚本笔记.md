[[脚本笔记]]是为 [[Obsidian]] 专门编排的“自动化脚本”，核心作用是帮助使用者摆脱重复性操作（比如重复输入固定格式、批量处理笔记内容等），将精力集中在更有创造性的笔记整理、内容创作上，提升 Obsidian 的使用效率。

Obsidian 的 [[Templater]] 插件有其原生的代码块格式，使用`<% %>` 单行模式  或 `<%- -%>`多行模式标注脚本代码的；而 [[NoteChain]] 插件则对这种方式进行了优化，不再使用原生标签，而是直接用 Obsidian 自带的「代码块」来编写和识别脚本笔记，操作更简洁、更贴合 Obsidian 的使用习惯。

## 一、NoteChain 脚本代码块的声明格式

在 Obsidian 中编写 NoteChain 脚本笔记时，需将代码块声明为以下任意一种格式（本质无区别，仅为方便记忆，任选其一即可），NoteChain 才能识别并执行该脚本：

- `js //templater`
- `js templater`
- `js //tpl`
- `js tpl`

说明：开头的 `js` 表示代码块的语言为 JavaScript，后面的注释是告诉 NoteChain：这是一个需要执行的 Templater 脚本代码块。

## 二、核心执行函数：parse_templater

NoteChain 提供了专门的 `parse_templater` 函数，用于执行编写好的脚本笔记。

使用者只需调用该函数，传入对应参数，即可实现脚本的自动化执行，无需手动操作。

### 1. 函数声明

函数的三个核心参数，分别对应不同的功能，具体解释如下：

- **template**：脚本的“来源”，有两种形式——要么是提前写好的「脚本笔记文件名」（直接调用整个笔记里的脚本），要么是直接写在代码块里的「脚本代码片段」；
    
- **extract**：布尔值（true/false），控制脚本的执行方式：
    
    - 当 extract = true 时：会自动识别笔记中所有标注为「js //templater」（及其他三种声明格式）的代码块，并执行这些代码块；
    - 当 extract = false 时：不识别上述代码块，而是按照 [[Templater]] 原生格式执行脚本。
        
- **extra**：额外需要传递的参数（可选），若有需要传递的变量、参数，可通过该参数传入，在脚本代码块中，用 `tp.config.extra` 即可访问到这些参数。
    

### 2. 函数调用示例

实际使用时，调用格式如下（固定写法，只需替换参数即可）：

```js
let rsp = await ea.tpl.parse_templater(template, extract, extra)
```

说明：`await` 是 JS 中用于等待异步操作完成的关键字，必须添加；`rsp` 是变量名，用于接收函数的返回值（可自定义，比如改为 `result`）。

### 3. 函数返回值说明

`parse_templater` 函数的返回值是一个「列表」（数组），列表中的每一个元素，都对应你传入的 `template`（脚本笔记）中的每一个代码块。

简单说：脚本笔记里有几个可执行的代码块，返回的列表就有几个元素，顺序与代码块在笔记中的顺序一致。

### Template 的两种使用模式

`template` 是函数最核心的参数，有两种使用模式，分别对应不同的场景，结合示例理解更简单：

### 1. 单行模式

适合编写简短的脚本（一行就能完成），脚本代码需用 `<% 脚本内容 %>` 包裹，此时需将 `extract` 参数设为 `false`（因单行模式采用 Templater 原生标签格式，无需 NoteChain 识别代码块）。

示例1：

```js
await ea.tpl.parse_templater('ea.cfile', false)
```

解释：`template` 是 `'ea.cfile'`（字符串形式，无需包裹），`extract=false`，此时函数不执行任何脚本，仅返回字符串本身，最终返回值为：`["ea.cfile"]`（一个只有一个元素的列表）。

示例2：

```js
await ea.tpl.parse_templater('<%ea.cfile%>', false)
```

解释：`template` 是 `'<%ea.cfile%>'`（用 Templater 原生标签包裹），`extract=false`，此时函数会执行标签内的 `ea.cfile` 脚本（`ea.cfile` 是 Templater 自带的脚本，功能是获取当前打开笔记的路径），最终返回值为：`["当前打开笔记的路径"]`（列表中是脚本执行后的结果）。

### 2. 段落模式

适合编写复杂的脚本（多行代码、多个代码块），此时无需使用 `<%- -%>` 标签，直接采用 [[NoteChain]] 支持的代码块格式（即前文提到的 `js //templater` 等四种声明格式），将脚本代码写入代码块即可。

说明：段落模式下，通常将 `extract` 参数设为 `true`，如此 NoteChain 会自动识别所有符合声明格式的代码块，依次执行，并将每个代码块的执行结果，按顺序存入返回的列表中。

关于段落模式下脚本块的返回值，需明确其获取顺序，优先级从高到低依次为：

1. 优先获取代码中通过 `return` 语句返回的值，若脚本块中存在 `return` 语句，该语句的返回值将作为当前脚本块的执行结果，存入返回列表；
    
2. 若脚本块中无 `return` 语句，则获取代码中 `tR` 变量的值，`tR` 的值将作为当前脚本块的执行结果，存入返回列表；
    
3. 若脚本块中既无 `return` 语句，也无 `tR` 变量，则当前脚本块的执行结果为空，返回列表中对应位置将存入空值。
    

结合前文函数返回值的说明，整个 `parse_templater` 函数的返回列表，会按脚本块在笔记中的顺序，依次存放每个脚本块按上述规则获取到的返回值，便于后续对执行结果进行调用和处理。
